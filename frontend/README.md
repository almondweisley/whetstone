# Whetstone Flutter web frontend

This Material-only Flutter web app implements the frozen Django contract in `../API_CONTRACT.md`. Its only non-SDK runtime package is `http`.

## Local development

Run Django in one terminal. The CORS origin must match Flutter's web port:

```bash
set -a; source .env; set +a
CORS_ALLOWED_ORIGIN=http://localhost:3000 POSTGRES_PORT=5433 python manage.py runserver 8000
```

Run the Celery worker in another terminal for queued runs to progress:

```bash
set -a; source .env; set +a
celery -A whetstone worker --loglevel=info
```

Then, from `frontend/`, start the web app:

```bash
flutter run -d chrome --web-port 3000
```

The compiled default API URL is `http://localhost:8000`. To use another API server, make it a compile-time value:

```bash
flutter run -d chrome --web-port 3000 --dart-define=API_BASE_URL=https://api.example.com
```

## Dart and Flutter map

- `lib/models.dart` has immutable Dart classes matching the API. Their `factory` constructors turn raw JSON maps into typed values, keeping JSON keys out of widgets.
- `lib/api_client.dart` owns all HTTP. A `Future<T>` is a result that arrives later; `await` waits for it without freezing the browser. `String.fromEnvironment` reads the compile-time API URL.
- `lib/main.dart` builds the screen. `WhetstoneApp` and candidate cards are `StatelessWidget`s because they hold no changing data. `GeneratorScreen` is a `StatefulWidget`; its state holds the form input, API data, and a two-second `Timer`. `setState` requests a redraw when data changes.
- `pubspec.yaml` lists the Flutter SDK and the single `http` package. Flutter resolves dependencies from this file.
- `web/` was generated by Flutter. It contains the HTML host page, manifest, icons, and browser configuration for a Flutter web build.
